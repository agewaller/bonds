## このセッションで確立したオーナーの方針（プロダクトの向き）

1. **連絡先リストの大半は「死んだリスト」である。** 全員を等しく見せず、重要と思われる方を
   自動でピックアップして、そこに関係を高める打ち手を集中させる。名寄せは基本自動化し、
   ユーザーの手を煩わせない（どうでもよい人・死んだリストが混ざっている前提）。
2. **相手の深い情報（近況・悩み・目標）は web には無い。** 情報は掘りに行くのではなく、
   やりとりの副産物として自然に溜まる仕組みを作り、取りこぼしをゼロにする
   （ユーザーの頭の中・手元のやりとり・相手からの返信・会社/地域/季節の輪郭情報の4源泉）。
3. **LINE トーク履歴の取込は正直大変。** 受動収集（Google 連携）と軽い還流（一行メモ・
   返信貼り付け・1日1問）を主役にする。
4. **関係には目標がある。** ビジネス・プライベート・婚活・恋活・家族関係など様々な用途で、
   目標の距離感を決め、差から接触ペースと次の一手を出す。どの用途でも相手の気持ちと
   ペースを尊重し、駆け引きは提案しない。
5. **画面のすべての項目は折りたためるようにする**（パネルが増えても自分仕様に整理できる）。
6. **検索は全員が対象**（名前・ふりがな・ローマ字・メール・電話・会社・メモ）。
